> **Last updated:** 2026-07-03 (v0.5.0: Claude Sonnet 5 pricing support. v0.4.0: plain-language, collapsible in-app metric explanations across Prompts / Skills / Subagents / Workspaces, a `<synthetic>`-row note, and a clarified Settings plan text; corrected Team plan monthly prices. v0.3.1: scanner per-content-block accounting fix) — see [recent commits](https://github.com/muckybuzzwoo/token-dashboard/commits/main) for the running change log.

## Differences from upstream

- The Skills route is renamed **Skills & Commands** and splits invocations into two distinct counts: "You ran" (distinct sessions where you typed `/slash-command`, tracked via Claude Code's native `attributionSkill` field on assistant turns) and "Claude invoked" (real `Skill` tool calls Claude emitted, typically from `Task`/`Agent`-dispatched subagents). Both sources are de-duplicated against each other so the same slash command never inflates both columns.
- Skills tab shows declared output-token budget vs measured p50 / p95, and flags skills that exceed budget by more than 20%.
- Overview-style endpoints (`/api/overview`, `/api/projects`, `/api/sessions`, etc.) read from materialised summary tables, so the dashboard stays responsive at hundreds of thousands of messages.
- Settings tab can switch the `.claude` folder at runtime.
- Prompts tab can export the current view to CSV or Markdown.
- Scanner uses batched `executemany()` inserts and a single temp-table-driven snapshot eviction pass per file (PR #18); a full rescan of ~600 files / 60k+ messages completes in well under 10 seconds on commodity hardware.
- Scanner accounts for the **current per-content-block transcript format** (v0.3.1, from the analysis in [nateherkai#25](https://github.com/nateherkai/token-dashboard/issues/25) by @sdkkds). Claude Code now splits one assistant response into disjoint records that share a `message.id` and each repeat the final `usage` (not growing snapshots). The snapshot dedup still collapses these to one message row (token totals match billing), but **re-points the evicted siblings' `tool_calls` onto the surviving keeper** (deduped by `tool_use_id`) instead of dropping them — so parallel `tool_use` blocks are no longer lost from Top Tools / Skills counts. Turn counts (`overview`, `projects`, `sessions`, summary tables) now filter `prompt_text IS NOT NULL`, excluding tool-result-as-user records that previously inflated "turns" ~6–8×. Verified token-neutral and idempotent across full rescans on real transcripts. Token/cost figures are unaffected. (Existing DBs need one rescan to pick up the recovered tool rows.)
- One-time CLI helpers: `python cli.py rescan-agent-targets` (back-fill `Agent` rows that were ingested before the `Task→Agent` rename) and `python cli.py rescan-slash-commands` (synthesise `Skill` rows from historical slash-command user messages — kept as a fallback for DBs that pre-date the `attribution_skill` column).
- The optional RTK savings tab probes `~/.local/bin/rtk` at request time and shows install instructions when absent; on Windows it always shows the install state because that path does not exist.
- Two attribution tabs from PR #22: **Workspaces** (bipartite Sankey of agent cwd → file target + cross-workspace leaks table) and **Subagents & Orchestration** (per-model main / Task-subagent / auto-compaction split, SDK-entrypoint runs, dispatcher → child dispatch tree). New tip rule fires when an agent in workspace X reads files in workspace Y > 50 times in 7 days. The upstream PR's hardcoded tier-routing skill panel (`orchestrate` / `agentic-efficiency-harness` / `pareto-cascade-delivery` / `run-benchmark`) was stripped because this fork's owner does not use those skills — the corresponding `tier_routing_stats()` query, `tier_routing_gap_tips()` rule, and the first KPI tile in the Subagents page are absent in this fork.
- Session detail page (`#/sessions/<id>`) header shows the **full** session UUID, not a truncated `4edcfece…` prefix — useful when copying the ID for bug reports or cross-referencing JSONL files. Listings (Sessions tab, Top sessions tables) keep the 8-character prefix to save horizontal space.
- **Plain-language, collapsible in-app explanations (v0.4.0)** — every data-heavy tab now carries a collapsible legend built on the same `<details>` glossary pattern as the Overview's "What do these numbers mean?": Prompts, Skills & Commands, Subagents (agent kind, entrypoint, dispatch tree, top sessions, per-(kind, model) table) and Workspaces. The previously dense, jargon-packed column/chart descriptions were rewritten into scannable *term → definition* lists with concrete examples and a "what you can read out of it" framing, collapsed by default with a "— click to expand" hint. Also adds a note explaining `<synthetic>`-model rows on the Prompts tab (Claude Code's auto-injected "Continue from where you left off." continuations after a resume / auto-compaction — they carry token counts but no price), and clarifies in Settings that picking a subscription plan never changes the displayed dollar figures (always the API pay-per-token value) — it only shows your flat monthly fee alongside for comparison.
- **Tips engine expansion (2026-05-28)** — 10 additional rule-based tip detectors layered on top of the existing cache / repeat-file / repeat-bash / right-size / tool-bloat / subagent-outlier / cross-workspace / skill-budget / claude-md-size rules:
  - `dead-skills` — installed skills with zero `Skill`-tool invocations in 90 days, excluding those installed in the past 30 days (file mtime as install-age proxy). Fires when at least 5 dead skills are found.
  - `subagent-sprawl` — sessions where sidechain (subagent) tokens exceed 2× main-chain tokens with >50k sidechain total in 7 days, excluding `acompact-*` auto-compaction.
  - `bash-bloat` — Bash commands averaging >5k tokens of output without an obvious output limiter (`head`, `tail`, `wc -l`, `Select-Object -First`, `-TotalCount`, …). Cross-shell heuristic. Requires the new `tool_calls.tool_use_id` column to join command rows to their `_tool_result` rows.
  - `context-pressure` — sessions whose peak per-turn net-new tokens (`input + cache_create_5m + cache_create_1h`, deliberately excluding `cache_read_tokens` because Anthropic multi-counts cache reads across breakpoints) exceeds 100k.
  - `bash-errors` — Bash commands that errored ≥3 times in 3 days with identical command text.
  - `web-fetch-volume` — sessions with ≥15 web-fetch calls in 7 days. Matches `WebFetch` and any `mcp__server__tool` whose name carries `fetch` / `read_url` / `scrape` / `crawl` / `browser_navigate` — no specific brand (Jina, Firecrawl, …) is hard-coded.
  - `opus-only` — projects where >90% of assistant turns in 14 days ran on Opus, with at least 50 turns to filter noise.
  - `mcp-sprawl` — ≥12 distinct MCP servers active in 7 days.
  - `claude-md-stack` — a cwd whose ancestry contains ≥3 CLAUDE.md files totalling >400 lines.
  - `long-skill-descriptions` — ≥3 installed skill descriptions exceed 400 chars.
- **Strict plugin-slug match in the skill catalog (2026-05-28)** — `_slugs_for()` in `skills.py` previously registered every non-structural ancestor path segment as a possible `<plugin>:<skill>` prefix. On Windows the home path leaked `Users` and the username into the catalog as fake plugin names; on either OS the marketplace directory name was indistinguishable from a real plugin. The catalog now anchors strictly on the two documented on-disk layouts (`marketplaces/<m>/plugins/<plugin>/skills/...` and `cache/<m>/<plugin>/<version>?/skills/...`) plus a plugin-root `SKILL.md` fallback when the parent directory matches a version pattern. The `skill_listing_budget_tips` candidate list dedupes by `SKILL.md` path and prefers the plugin-qualified form for display.
- **`tool_calls.tool_use_id` column (2026-05-28)** — additive schema migration linking each tool_use row to its matching `_tool_result` row via the `tool_use_id`. Required for `bash-bloat` and `bash-errors` per-command attribution. Migration auto-runs on next `init_db()` (server start) and is idempotent. As with prior migrations, the data tables (`messages`, `tool_calls`, `files`, `summary_meta`) are cleared so the next scan replays JSONLs with the new column populated — source of truth lives on disk, full rescan is ~10 seconds on commodity hardware.
- **Scope-aware skill catalog (2026-05-28)** — `scan_catalog()` previously rglobbed every `SKILL.md` under `~/.claude/plugins/`, including marketplace clones (downloaded plugin metadata that was never installed). Those phantoms inflated the `skill-budget`, `dead-skills`, and `long-skill-descriptions` tips. The catalog now reads `~/.claude/plugins/installed_plugins.json` and only scans the `installPath` of each currently-installed plugin; user-skills (`~/.claude/skills/`), scheduled-tasks, and project-local `.claude/skills/` discovered from `messages.cwd` are still included. Each catalog entry now carries a `scope` (`user-global` / `project-global` / `project-local` / `unknown`) and a `project_path`, so tips can reason per-session: `skill_listing_budget_tips` now reports the **effective per-session footprint** intersected with the user's most-active recent cwd, splits "N global skill(s) plus M project-scoped skill(s) load per session" in the body, and names the project. `dead_skills_tips` skips project-scoped skills whose `projectPath` had zero recent activity (those zero invocations are meaningless — the user simply hasn't worked in that project). Legacy blanket scan still fires when the manifest is missing/unreadable, but **not** when it is valid-but-empty (the user has uninstalled everything; clones must stay excluded). Bounds-check on `installPath` rejects filesystem roots to neutralise a tampered-manifest rglob DoS. Live impact on a heavy user's machine: catalog 448 → 208 slugs, skill-budget tip reports ~18.9k effective chars instead of inflated all-on-disk counts.
- **`enabledPlugins` + skill-visibility filtering in budget tip (2026-05-29)** — `_read_installed_plugin_entries()` now reads `~/.claude/settings.json`'s `enabledPlugins` dict and skips plugins whose entry is `false` (installed but disabled). `skill_listing_budget_tips()` now skips skills with `disable-model-invocation: true` in their frontmatter, or whose slug is set to `user-invocable-only` / `name-only` / `off` via `skillOverrides` in global or project `settings.local.json`. Both changes prevent false-positive budget warnings for skills the model never actually loads. `scan_catalog()` replaces `Path.rglob()` with `_iter_skill_files()` using `os.walk(followlinks=True)` + cycle detection so symlinked project skill directories are correctly discovered. Community [PR #1](https://github.com/muckybuzzwoo/token-dashboard/pull/1).
- **Prompts tab re-linked to assistant turns (2026-06-10)** — a newer Claude Code version changed the transcript format: it now writes `attachment` / `queue-operation` / `last-prompt` records and re-parents the assistant turn through an `attachment` node, so `assistant.parent_uuid` no longer points at the typed prompt (and assistant rows carry no `prompt_id`). The old `expensive_prompts` join (`JOIN messages a ON a.parent_uuid = u.uuid`) silently dropped to **0 matches** from ~late May onward, so the Prompts tab appeared frozen at 2026-05-29 and never showed Opus 4.8 (the model used from June). The DB and every aggregate view (Overview, per-model, Projects, Sessions) were always correct — only this one join broke. Rewrote `expensive_prompts` to link by **session + time window** instead of `parent_uuid`: take the typed prompt per `(session_id, prompt_id)` (earliest row — collapses the injected `list[text]` skill/system messages that share the prompt's `prompt_id`), main-thread only (`COALESCE(is_sidechain,0)=0`), and attribute every main-thread assistant message between it and the next prompt. `billable_tokens` is now the full turn's cost (incl. its tool loop), not just the first response. Pure query change — no schema migration or re-scan needed. Regression tests cover the `attachment` interposition and the shared-`prompt_id` collapse.
- **Prompts tab: NULL-model first assistant no longer drops the prompt (2026-06-15, v0.2.1)** — follow-up correctness fix to the session+time-window rewrite above. The displayed-model subquery picked the *earliest* assistant in the window with no `model IS NOT NULL` guard; combined with the outer `WHERE model IS NOT NULL` filter, a leading API-error / synthetic assistant row (`model` NULL — written by the scanner whenever `message.model` is absent, e.g. an error turn before a successful retry) caused the **entire** prompt to be dropped from the Prompts tab, including the billable tokens of the real response that followed. Fixed by skipping NULL-model rows in the model-pick subquery only (`... WHERE {win} AND a.model IS NOT NULL ...`); the token SUMs still span the full turn, so an error row's (usually zero) tokens are still counted. Regression tests added for the NULL-model-first ordering and for multi-assistant (tool-loop) summing. Pure query change — no schema migration or re-scan needed.
- **Fable 5 model support (2026-06-10)** — adds `claude-fable-5` to `pricing.json` as a new tier **above** Opus at **$10 / $50** input/output per MTok (cache rates $1.00 read / $12.50 5m-write / $20.00 1h-write, derived from Anthropic's standard 0.1× / 1.25× / 2× input multipliers). The tier matcher (`_tier_from_name` in `pricing.py`), the frontend badge classifier (`fmt.modelClass` in `web/app.js`), and a new `.badge.fable` style (gold, distinct from Opus-purple / Sonnet-blue / Haiku-green) all learned the `fable` tier, and a `fable` `tier_fallback` entry prices future dated/suffixed variants instead of returning a null cost. Opus 4.8 needed no work — it was already integrated via community [PR #2](https://github.com/muckybuzzwoo/token-dashboard/pull/2) and Claude Code persists it to the JSONL as the bare `claude-opus-4-8` (the `[1m]` context annotation is not written to transcripts), so it already matches the exact-pricing entry.
- **Sonnet 5 model support (2026-07-03)** — adds `claude-sonnet-5` to `pricing.json` at the standard Sonnet-tier list price (**$3 / $15** input/output per MTok; cache rates $0.30 read / $3.75 5m-write / $6.00 1h-write), identical to `claude-sonnet-4-6`. Unlike the Fable 5 addition this needed **no new tier plumbing** — the `sonnet` tier already exists in `_tier_from_name` (`pricing.py`), the `fmt.modelClass` badge classifier (`web/app.js`), the `.badge.sonnet` style, and the `sonnet` `tier_fallback` — so the only effect of the exact `models` entry is reporting `estimated: false` instead of the tier-fallback's `estimated: true`. Anthropic's current **$2 / $10 introductory rate (through 2026-08-31) is intentionally not encoded**: `pricing.json` has no time-windowed pricing, all entries use list prices, and an intro rate would be wrong after the promo window.
- **Local-timezone time display (2026-05-29)** — all timestamp rendering now uses the **browser's local timezone** instead of a hardcoded `America/New_York` (inherited from upstream). `fmt.ts` (Sessions list/header, Prompts, Subagents, Skills) dropped the `timeZone: 'America/New_York'` override; a new `fmt.time` helper renders the Sessions turn-by-turn time column, which previously showed **raw UTC** (`timestamp.slice(11,19)`) and so disagreed with the session header. Both now use `toLocaleString` / `toLocaleTimeString` with the `'sv'` locale for ISO-style `YYYY-MM-DD HH:MM:SS` formatting. The turn-by-turn `data-val` keeps the raw UTC ISO string, so the click-to-sort key stays chronological across day boundaries. The RTK "Today" card's `today()` and the Prompts CSV filename stamp likewise switched from `America/New_York` / UTC to browser-local. **Deferred:** server-side daily-chart bucketing still groups by **UTC** calendar day (`substr(timestamp, 1, 10)` in `db.py`), so a turn after local midnight can land on the previous day's bar; converting that to local-day bucketing (touches the `idx_messages_date` index, `_date_range_clause`, and the bundle cache key) is a separate change not done here.
